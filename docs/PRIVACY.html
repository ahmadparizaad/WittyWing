1. Data we collect

The extension is intentionally minimal: it only stores a Google Gemini API key (provided by you) and temporarily reads tweet text from the page when you request a generated reply.

We explicitly declare the following data types used by the extension:

  • Website content: Tweet text from the page. Sent to Google Gemini only when you click Generate.
  • Authentication information: Your Google Gemini API key stored locally in chrome.storage.local to authenticate requests to the Gemini API.

Gemini API key: The extension stores your Google Gemini API key locally in the browser extension storage (chrome.storage.local). This key is used only to call Google Gemini APIs and is not transmitted to any developer-controlled server.

Tweet text: When you click the Generate button, the visible tweet text (and any optional tone selection) is sent to Google Gemini as part of the prompt to generate a suggested reply. Tweet text is transmitted only on explicit user action and is not collected continuously.

2. How we use your data

We use the stored Gemini API key to authenticate requests to Google’s Gemini API and the tweet text to generate a suggested reply. Generated replies are displayed in your browser UI and may be copied to your clipboard if you use the Copy action.

The extension does not keep a history of tweet text on external servers. Generated text is not sent to developer servers.

3. Data sharing

We do not share your Gemini API key or tweet text with third parties other than Google (the API provider) when making API calls to Gemini. The extension does not send analytics, logs, or other user data to external servers.

4. Security

The Gemini API key is stored locally in chrome.storage.local. Only the extension code can access this storage. Do not share your API key with untrusted parties. We recommend using a dedicated API key with minimal permissions and monitoring your Google Cloud usage and quotas.

All network requests to Google Gemini are made over HTTPS. API calls use a 30-second timeout to prevent long-running requests. The extension does not fetch or execute remote JavaScript or WebAssembly at runtime.

5. Removing your data

To remove your API key, open the extension popup and delete the key or uninstall the extension. You can also clear extension data from Chrome's settings. Uninstalling the extension removes local extension storage.

Exact steps to delete the stored key:

  1. Click the extension icon in Chrome to open the popup.
  2. In the popup, clear the API key input field and click the Save/Remove button (or click the provided delete action).
  3. Alternatively, go to chrome://extensions → Details → Site storage and clear data for this extension.

6. Contact

If you have questions or issues, please open an issue in the project repository or contact the developer via the repository page.

Preferred contact: open an issue at https://github.com/ahmadparizaad/twitter-automation/issues

7. Changes to this policy

We may update this policy from time to time. Significant changes will be indicated by updating the "Last updated" date at the top of the page.
